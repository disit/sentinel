run_single_crawl.sh takes 4 arguments; command, log path, hours before considering a log old, and optionally a time delta in hours in case timestamps are off due to timezones shenanigas

first it runs the command
if the command has a failure, it is logged as such
otherwise (exitcode 0) it didn't fail
in either case the time when it happened is stored in a separate log (respectively for failure and success)

then it searches the produced log in log path (the parameter), note that it is expected for this to exist and be populated by the command ran, as seen in the examples at end of this file
if it finds an endtime (it is expected it to be on the last line at the end of that line) it is matched against current time
the tldr is if it is too old, we somehow didn't update, we are reading an old one, and the old one is too old (as for 3 parameter)

then we check if the latest failure happened more recently than latest success, if it did (or success never happened) then an error is printed
